七、插槽

Vue 中组件的标签有两种写法。

第一种是自结束标签，第二种就是一般的标签。Vue 专门保留了两种写法，说明这两种写法都有其自己的作用。前者就是一般的组件标签，而后者有额外的功能——插槽。

Vue 的插槽功能允许向组件标签的内部再追加内容，并且浏览网页时能在子组件中显示。插槽一般用于有多个同类的子组件实例，但每个实例中的显示内容又不一致的情况。

八、组合式 API

组合式 API 是 Vue 3.0 新提出来的东西，因此 Vue 2.0 是不适用的。

(1) emits

emits是 Vue 3.0 新追加的一个应当被定义在 Vue 组件的配置对象中的属性。该属性用于接收来自父组件传递给子组件的自定义事件。如果没有使用emits属性 “注册” 自定义事件，那么自定义事件就将对子组件不可用。

emits属性应当是一个数组，其存储的值是String类型，标记要 “注册” 的自定义事件的名称。

(2) setup()

之前所写的data()、methods等，其实都是不规范的（但仍能使用）。在 Vue 3.0 中，数据、方法、计算属性、生命周期钩子、监视属性等全都应该写在setup属性中。

setup()属性应该有一个返回值，这个返回值是对象，它的属性在 HTML 模板中可以直接使用，而没有返回的数据则无法直接使用，但可以通过闭包使用。但请注意，上述写法无法做到响应式。

另外，setup()函数执行的时机是在beforeCreate()回调函数执行之前，并且只会执行一次。setup()函数会在执行时接收到两个参数，第一个参数是使用props属性接收到的、从父组件传递过来的参数的对象的代理对象（关于什么是代理对象，下文将详述）；第二个参数则是一个上下文对象。该对象含有一个emit()方法，用于触发自定义事件。关于这个对象此处不做过多赘述，详情请参见 Vue 3 官方文档 。

(3) ref()

这个ref()与之前提到的与 HTML 元素的id属性类似的ref属性不同。当然ref属性并没有被删除，这里讲的是一个在 Vue 组件配置对象中的ref()函数。

经过ref()函数加工的数据可以实现响应式，但修改其值的时候不能直接修改变量，而是应该修改这个变量的value属性，因为ref()函数返回的是一个对象，该对象是引用实现的实例（简称引用对象）。它的底层实现原理依旧与数据代理一致。

在 HTML 模板中使用ref()函数加工过的数据时，不需要加value属性，Vue 会自动帮你解析。

(4) reactive()

相比起ref()函数，通过reactive()函数绑定的对象更加啊便于使用，但reactive()函数不能绑定基本类型的数据，只能绑定对象。

其实ref()函数的底层也是调用了reactive()函数来绑定对象数据。reactive()函数可以实现更轻量级的深度监视，即不管对象套娃套了多少层，它都可以做到监视。它也可以用于绑定数组。

(6) Reflect

深入了解过 C#或者 Java 的同学应该已经学过反射了。

Reflect是一个定义在window对象下的对象，它与 “静态”的 Object对象有点相似（因为 JavaScript 标准的设计者们在尝试把一些Object 对象下的方法转移到Reflect对象下）。

Reflect对象下常见的方法有：

方法名 参数 功能 返回值
get() 对象，属性名 获得该对象的指定属性值 该对象的指定属性值
set() 对象，属性名，值 将该对象的属性的值修改为指定值 boolean ，指示操作是否成功
defineProperty() 对象，属性名，配置对象 设数据置代理 boolean ，指示操作是否成功
deleteProperty() 对象，属性名 删除该对象的指定属性 boolean ，指示操作是否成功
Reflect.defineProperty()与Object.defineProperty()的区别
前者在重复定义属性时不会引发报错，只会返回false来指示修改不成功；而后者将会直接抛出异常，中断程序运行。

深度监视模式与监视reactive()创建的代理对象
深度监视模式在监视代理对象本身的情况下是失效的（或者说没法关闭），因为Vue 3.0的底层代理逻辑（ Proxy ）就注定了它只能是深度监视的。另外，监视代理对象时，虽然能侦测到属性值的变化，但是传给回调函数的New与Old都是代理对象本身，即无法获取改变之前的值。因为传回的都是代理对象本身，它不掌握数据，所以新旧值都是引用的外部数据，引用本身不变，而这个外部数据已经被修改了。

2. watchEffect()

watchEffect()函数是Vue 3.0新增的函数，该函数同样也能用于监视。它只接受一个参数，该参数是一个回调函数。

使用watchEffect()函数进行监视时，不需要指明要监视的变量。所有在该回调函数内使用的变量的值改变时，回调函数就会重新执行。相比watch()函数，watchEffect()函数智能了不少。通常会在该函数的回调函数内编写Ajax请求。

(9) toRef()与toRefs()

toRef()函数接受两个参数，前者是一个对象，后者是该对象的属性名。该函数会将指定的属性加工为引用对象。使用该函数时可以传入reactive()函数创建的代理对象，生成的引用对象也具有响应式的效果。

它与ref()函数最大的区别在于，前者是真的 “引用” ，而后者是新建实例的 “引用” 。当传入的数据是一个代理对象时，修改前者创建的引用对象，代理对象也会跟着改变；而后者无法做到这一点，因为后者 “引用” 的是一个全新对象。

而toRefs()则是一次性处理一个对象下的所有属性，因此它不需要第二个参数。

反正我是觉得这俩函数花里胡哨的没啥用。
